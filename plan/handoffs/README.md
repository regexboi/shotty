# Handoffs

Write one handoff document per completed phase:

- `phase-1-handoff.md`
- `phase-2-handoff.md`
- `phase-3-handoff.md`
- `phase-4-handoff.md`
- `phase-5-handoff.md`

Each handoff should include:

- What was completed
- What remains for the next phase
- Any blockers, bugs, or compromises
- Files and areas of the codebase that matter most next
- Exact verification status
- Anything the next agent should be careful not to break
